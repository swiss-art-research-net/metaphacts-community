SARI Components

The platform provides a set of components developed by SARI, the Swiss Art Research Infrastructure at the University of Zurich, Switzerland. All components developed by SARI are prefixed with sari-*.

Copy query to clipboard

The <sari-copy-query-to-clipboard> components can be used to copy the query provided in the `query` attribute to the clipboard. If used within a `semantic-search-result` component, it can be used to allow users to copy the generated query to the clipboard.

File Upload

The <sari-file-upload> implements a file input field alongside a template that receives the uploaded file object. The content of the file is available in base64 encoding. The component works on client-side without a backend.

The component implements the `template` and `no-result-template` attributes. The latter is displayed before a file has been selected, the former can be used to work further with the uploaded file.

The default attributes of the file input field (accept, capture) are supported. A maximum accepted file size in MB can be specified through the `max-file-size` attribute.

Image Upload

The <sari-image-upload> component is an extension of the <sari-file-upload> component. Its `accept` attribute is pre-set to images, but can be overwritten. In addition to the features provided by the <sari-file-upload> component, the <sari-image-upload> accepts a `max-pixels` attribute that accepts a number. If present, the uploaded image will be scaled down to the given size in pixels. The scaling happens on the client side.

Resource Thumbnail

<sari-resource-thumbnail> is an alternative to the <mp-resource-thumbnail> component. In addition to the default behaviour of displaying a thumbnail for a resource, it also provides the possibility to use the thumbnail IRI in a template. The latter lends itself to displaying additional information about a thumbnail, such as license information.
