Template Page Editor

The Template System is the basis for rapid semantic applications development in the platform. A core part for the app development experience is the visual template editor.

As of the 4.0 release the platform integrates a state-of-the-art editor (based on the editor that powers Microsoft Visual Studio Code). This allows us to provide a smooth editing experience with intelligent features such as on-hover information, auto-completion and syntax validation.

The individual features are presented in more detail below.

Editor features

Feature: on-hover info for tags and attributes

The language service provides tag and attribute descriptions for basic HTML tags and platform components.

To access this infortmation, move the mouse over a component tag or any attribute, or alternatively place the text cursor inside and press Ctrl/Cmd+K then Ctrl/Cmd+I.

Feature: link to the component documentation

For platform components the language service provides links to their respective documentation pages. This information can be accessed when moving the mouse over a component tag.

Feature: jump to the included template page

The language service marks included template references as links and allows to jump to them directly from the editor.

To use this feature, hover the mouse over any template reference or set text cursor inside and press Ctrl/Cmd+Click.

Feature: tag completion

A very powerful feature is that of auto-completion: the language service auto-completes basic HTML tags as well as platform component by pressing Ctrl/Cmd+Space (while typing).

Feature: attribute completion

Similarly to tag completion, the language service can complete attributes of platform components.

To use this feature, start typing attribute or press Ctrl/Cmd+Space while typing.

Feature: JSON syntax highlighting in attributes

The template language defines custom syntax highlighting which recognizes embedded JSON in attribute values.

Feature: display tag and attribute warnings (experimental)

The visual editor supports the template developers with syntactical validation checks. This feature is disabled by default and can be can be enabled manually by toggling the "Show warnings" switch.

The following types of warnings are currently available:

  • unclosed non-void HTML tags
  • required but missing attributes
  • unknown (mistyped) attributes
  • incorrect datatype for attribute value
  • invalid JSON syntax inside attribute values

Feature: HTML formatting (experimental)

The language service supports formatting of selected text or the whole template document.

This feature is considered experimental: only pure HTML and client-side templates are supported and formatting may or may not work with backend Handlebars syntax.

To use this feature, open context menu for selected text and choose "Format Selection" or "Format Document", or press Alt/Cmd+Shift+F to format selected text or Ctrl/Cmd+K then Ctrl/Cmd+F to format whole document.
