Loading the data

First, please download the sample data (simple right click and save as). You can now load this data into the database via the data import and export administration page. The data will be stored in a dedicated named graph.

Searching & exploring the data

metaphactory offers functionality for searching, exploring and inspecting data.

In the following we describe how the data that has previously been loaded (see above) can be visually explored.

Inspecting the named graph

On the data import and export page a table lists the named graphs available in the database. This table should also show a link to the graph containing the sample data, which is named using the scheme file:///sample.ttl-%timestamp%. Following this link leads to a page for the respective named graph, offering different visualization views of the data:

  • Statements (default): the raw RDF statements contained in this graph
  • Graph: a graph visualization of the data
  • HTML / Page view: statistics about the named graph

The following figure depicts a graph visualization of the RDF resources available in the sample data. For instance, we can see three persons (Bob, Alice, and Leonardo da Vinci) and the painting Mona Lisa. The graph visualization provides a good overview of the data and shows the relationships between the resources.

Note that using the top-right toggle buttons (marked with 'View Selection') the view can be switched.

Exploring resources

From the Graph View the individual resources can be explored. By focusing Mona Lisa node on the graph and then clicking the icon that appears to the left of the node we navigate to the page of Mona Lisa.

The metaphactory offers a unified view for all resources. On the page of Mona Lisa we see again the three toggle buttons located at the top-right corner for view selection

From left to right:

  • Graph View - for a visual graph representation of the data enabling navigation through the knowledge graph.
  • Statements View - to display the raw data for the current resource as statements in tabular form.
  • HTML / Page View - to provide a domain specific end-user view, defined through a template for a particular class. If there is no template defined yet, this view is intentionally left blank.

Go ahead and switch to Statements View to learn that Mona Lisa was created by Leonardo Da Vinci. You can now click on Leonardo Da Vinci which will immediately take you to his resource page.

Querying the data

You can also query the data via the SPARQL endpoint. For example, you may want to issue a query for all statements made about Bob as a subject:
SELECT * WHERE { <http://example.org/bob#me> ?predicate ?object }
Simply copy the query and paste it into the editor field of the SPARQL endpoint, then click Execute.

Creating a template

Now, we want to create a template for resources of a particular class. The general templating mechanism is described here . In our example, we create a template for the class foaf:Person. For every person, we want to display:
  • the person's name
  • an image, if available
  • his interests
  • his friendship relationship
To edit a template page, you have two choices:
  • either go the template page directly, i.e. Template:foaf:Person

Inserting and updating data

Next, we want to create and add new instance data. We will do so via a SPARQL UPDATE operation against the SPARQL endpoint.
